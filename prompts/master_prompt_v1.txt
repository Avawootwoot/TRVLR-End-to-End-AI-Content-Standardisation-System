3. HTML PROCESSING RULES
The source text may contain HTML from booking systems.
Before rewriting, clean the HTML as follows:
Remove ALL HTML tags
 (<p>, <br>, <div>, <ul>, <ol>, <li>, <span>, <h1>, etc.)


Convert structure into clean text:


<p> → paragraph break


<br> → line break


<ul>/<ol> → list grouping


<li> → bullet point (“- ”)


Remove attributes, e.g. data-start="", style="", scripts, or embedded CSS


Decode HTML entities (&nbsp;, &amp;, etc.)


Collapse repeated spaces or blank lines


Ignore hyperlinks


After the HTML is cleaned and flattened into readable source text, rewrite using all
style and structure rules below.

MULTI-DAY LOGIC RULE
IF THE PRODUCT IS MULTI-DAY
A product is considered multi-day if the cleaned source text contains phrases like:
“Day 1”, “Day 2”, “5-day tour”, “multi-day”, “package”, “bundle”, “itinerary”, or clear references to multiple sequential days.
THEN the Description section must include:
1. A Multi-Day Overview Paragraph
Before listing days, begin the Description with:
a polished overview of the entire multi-day experience
a summary of what is covered across the full itinerary
a cohesive “journey flow” (start → highlights → finale)
2. A Day-By-Day Itinerary Breakdown (inside Description section)
After the overview paragraphs, include a structured itinerary:

Format:
### Itinerary

**Day 1: [Title]**  
[Short description of Day 1 activities]  
⏰ [Times if provided]  
✔ [Inclusions]  
✖ [Exclusions]  

**Day 2: [Title]**  
[Short description]  
⏰ …  
✔ …  
✖ …

(repeat for however many days appear in source text)

Rules:
Do NOT invent content. Use only information from the source text.
Keep each day concise, polished, and easy to scan.
Times (if provided) must be formatted like: ⏰ 7:00 am – 6:00 pm
Inclusions/exclusions must use ✔ and ✖
Use markdown, not HTML
3. A Closing Paragraph
End the Description with a warm, encouraging sentence summarising the entire multi-day experience.
IF THE PRODUCT IS SINGLE-DAY
Follow the standard 3–4 paragraph Description format.
B. Short Description Rules
Craft a single sentence, under 18 words, following this framework:
Emoji + 2–3 word intro + condensed attraction name + 4–8 word teaser
Guidelines:
● Clearly convey what it is and what you do
● Keep it tight, appealing, and natural
● Do NOT use emojis.
● Output must be plain text wrapped in HTML tags only (e.g., <div>...</div>).
Example:
‘Cruise to Green Island on this half-day snorkel tour to discover vibrant coral and
colourful tropical fish.’
C. Highlights
Provide 3–6 bullet points.
Rules:
Each bullet describes a clear feature or moment
Use sentence case
No full stops
No sales language
Keep each sentence short and to the point
Bullet example format:
Guided walk through ancient rainforest trails
Scenic views from elevated lookouts
Small-group experience with expert commentary
D. Inclusions Rules
List only what the source text explicitly mentions.
 Allowed items:
Equipment provided
What travellers need to bring
Safety notes
Age limits
Accessibility details
Vehicle types / gear
Necessary conditions or requirements
Bullet points only.
Do not assume or invent any inclusions.

E. Additional Information 
The Additional Information field MUST be populated.
Include 1–3 concise, factual bullet points that help travellers understand access, setting, conditions, or context of the attraction.
These points:
May rephrase or summarise facts already mentioned elsewhere


Must be derived directly from the source text


Must add planning or contextual value


Must NOT introduce new information


Suitable content includes:
Access method or pathway type


Terrain or environment


Wildlife presence


Visit conditions or general suitability


Conservation or natural setting context


Rules:
Bullet points only


Do not copy full sentences from other fields


Do not include dates, schedules, pricing, or availability


Do not invent information


This field must not be left empty
